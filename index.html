ISO 27001 for Hybrid Cloud

A comprehensive guide to essential controls for organizations migrating to a hybrid cloud infrastructure while maintaining ISO 27001 certification.

Organizational Controls

5.1 Policies for information security

Update policies to address cloud usage, data classification for the cloud, and shared responsibilities.

5.2 Roles & Responsibilities

Clearly define who manages, secures, and monitors the on-premises and cloud environments.

5.7 Threat intelligence

Collect and analyze threat intelligence for both your on-premises systems and the specific cloud services you use.

5.9 Inventory of information

Extend your asset inventory to include cloud resources, data, and their owners.

5.14 Information transfer

Establish secure transfer rules for moving data between on-premises and cloud environments.

5.23 Use of cloud services

Establish processes for acquiring, using, managing, and exiting cloud services in line with security requirements.

5.24 Incident management

Update incident management plans to include the hybrid environment and define response procedures.

5.30 ICT readiness for BCP

Update and test business continuity plans for the hybrid model to ensure service restoration.

5.31 Legal & regulatory

Identify and document legal requirements applicable to data in the cloud, such as data sovereignty.

People Controls

6.3 Awareness training

Train all personnel on updated security policies for the hybrid environment and cloud-specific risks.

6.7 Remote working

Implement robust security measures to protect information accessed from outside the organization's premises.

Physical Controls

7.1 Physical perimeters

Maintain responsibility for the physical security of your on-premises infrastructure.

7.8 Equipment siting

On-premises equipment connecting to the cloud must be securely sited and protected.

7.11 Supporting utilities

Protect on-premises facilities from power failures and other utility disruptions.

7.12 Cabling security

Secure physical cabling for dedicated connections to the cloud to prevent interception or damage.

Technological Controls

8.1 User endpoint devices

Protect information on all user devices (laptops, phones) that can access the hybrid environment.

8.5 Secure authentication

Implement strong authentication (e.g., MFA) for accessing both on-premises and cloud resources.

8.9 Configuration management

Establish and maintain secure configurations for all hardware, software, and services in the hybrid environment.

8.12 Data leakage prevention

Apply measures to systems and networks to prevent sensitive data leakage from any part of the hybrid environment.

8.13 Information backup

Adapt backup policies to ensure data in both on-premises and cloud locations can be restored.

8.16 Monitoring activities

Continuously monitor networks and systems across the entire hybrid environment for anomalous behavior.

8.21 Security of network services

Identify, implement, and monitor security for all network services, including connections to the cloud.

8.22 Segregation of networks

Use VPCs, subnets, and firewalls to isolate sensitive systems and control traffic flow in the hybrid network.

8.24 Use of cryptography

Define and implement rules for using cryptography to protect data at rest and in transit, with robust key management.

8.32 Change management

Apply formal change management procedures to all changes across the entire hybrid infrastructure.

This presentation is a comprehensive guide and should be adapted to your organization's specific context and risk assessment results.
